# v1.3 端到端验收清单（部署 + 联调）

目标：按 v1.3 需求逐项可复现通过，并能快速定位问题层级（前端/网络/后端/数据库）。

## 0. 变量准备
- 域名：`https://<你的域名>`（通过 Cloudflare Tunnel 访问）
- 后端本机端口（NAS 本机）：`http://127.0.0.1:8031`
- 必填环境变量：
  - `CLOUDFLARED_TOKEN`
- 建议环境变量：
  - `ACCOUNTING_SETUP_KEY`：随机字符串
  - `ACCOUNTING_CORS_ORIGINS`：前端访问域名，例如 `https://acc.example.com`

## 1. 后端容器健康与落盘
- [ ] Container Manager 项目启动成功，无反复重启
- [ ] 本机访问 `http://127.0.0.1:8031/health` 返回 `{"status":"ok"}`
- [ ] 数据落盘（至少能看到 `accounting.sqlite3`；若有 WAL/SHM 也正常）
- [ ] 日志落盘（`/logs/app.log` 有输出且按天滚动）

## 2. Cloudflare Tunnel 与 HTTPS
- [ ] 外网访问 `https://<域名>/health` 返回 `{"status":"ok"}`
- [ ] NAS 本机端口未对公网暴露（仅 `127.0.0.1` 绑定）
- [ ] 若使用自建前端域名：确认浏览器地址栏为 HTTPS

## 3. CORS（仅当前端与后端不同源时必测）
- [ ] 后端环境变量 `ACCOUNTING_CORS_ORIGINS` 填写为允许的前端域名
- [ ] 浏览器控制台无 CORS 报错
- [ ] 非白名单域名请求后端时被浏览器拦截（符合预期）

## 4. 首次注册/登录（邀请码）
首次仅当服务端还没有任何用户时成立：
- [ ] 前端“使用邀请码注册” -> “获取初始邀请码（仅首次）”成功
- [ ] 注册并登录成功
- [ ] 再次注册时返回 `single_user_already_exists`（符合 v1.3 单账号模式）

## 5. 登录态与 CSRF
- [ ] 未登录访问受保护接口返回 401（例如 `/sync/version`）
- [ ] 登录后访问 `/auth/me` 返回用户名
- [ ] 写接口缺少 CSRF 返回 403（例如 `/sync/push`）
- [ ] 写接口带 `x-csrf-token` 后成功

## 6. 同步（增量 + 幂等）
- [ ] 设置页点击“立即同步”成功，显示拉取/上报与版本号
- [ ] 连续点击“立即同步”不产生异常（同步调度器会合并/退避）
- [ ] 重复上报同一批数据（幂等 key）不会重复插入（applied 不持续增长）
- [ ] 服务端执行 `/sync/compact` 后，老客户端拉取触发 baseline_required，并能自动用 `/sync/baseline` 恢复

## 7. 回收站（删除/恢复/清空）与跨端一致性
- [ ] 删除交易记录后，正常列表不可见
- [ ] 设置页回收站能看到该记录
- [ ] 恢复后回到正常列表
- [ ] “彻底删除”清空回收站后不可恢复，并在其他设备同步后同样不可见

## 8. 回收站 30 天自动清理（服务端定时 + 同步传播）
两种验证方式（任选其一）：

### 方式 A（真实等待验证）
- [ ] 等待服务端定时任务触发（默认每天凌晨 3 点）
- [ ] 观察日志有 `auto_recycle_purge_ok`
- [ ] 其他设备同步后，本地回收站同样减少（收到 `purge_deleted_before`）

### 方式 B（快速验证）
- [ ] 临时将环境变量 `ACCOUNTING_RECYCLE_RETENTION_DAYS` 设置为 `0` 并重启后端
- [ ] 手动调用一次 `/sync/compact`（触发定时任务同一轮里也会执行清理）
- [ ] 观察回收站是否被清掉（并能通过同步传播）

## 9. 备份与恢复演练
- [ ] 停止项目后备份 `accounting.sqlite3*`（含 wal/shm）到备份目录
- [ ] 清空数据后用备份文件覆盖恢复
- [ ] 启动项目后能正常登录并同步
