# v1.3 本地联调指南（后端 + 前端）

## 什么时候开始做联调最合适
建议从以下 3 个“会跨端的能力点”开始联调（每个点都能独立验证，并能快速定位问题在哪一层）：
- 登录/注册（邀请码）闭环：前端能拿到 Cookie，会话能保持
- “立即同步”闭环：前端能 pull/push，且同步版本号递增
- 回收站闭环：删除进入回收站、恢复、清空后不可恢复，并能通过同步传播

当前代码已经具备上述 3 个能力的端到端路径，所以你现在就可以开始联调。

## 前置条件
- Node.js（用于启动前端 dev server）
- Python 3.12+（用于启动后端 FastAPI）

## 启动后端（开发模式）
在目录 `AccountingApp/backend` 执行：

```powershell
$env:ACCOUNTING_COOKIE_SECURE='0'
$env:ACCOUNTING_DB_PATH='./data/dev.sqlite3'
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8031
```

检查后端是否起来：
- `GET http://127.0.0.1:8031/health` 返回 `{"status":"ok"}`

## 启动前端（开发模式）
在目录 `AccountingApp` 执行：

```powershell
npm install
npm run dev
```

前端 dev server 通过代理访问后端（默认指向 `http://127.0.0.1:8031`），你也可以用环境变量指定：

```powershell
$env:VITE_BACKEND_URL='http://127.0.0.1:8031'
npm run dev
```

## 首次注册/登录流程（邀请码）
首次只有“引导邀请码”能注册（仅当服务端还没有任何用户时成立）：
1. 打开前端页面后，切到“使用邀请码注册”
2. 点击“获取初始邀请码（仅首次）”
3. 设置用户名/密码，点击“注册并登录”

注册成功后，再次注册会被拒绝（v1.3 单账号模式）。

## 立即同步验证（最短路径）
登录后进入“账户”页：
- 点击“立即同步”
- 预期看到同步结果：拉取/上报条数与当前版本号

如果看到 “baseline_required”，前端会自动去拉取基线并重建本地数据，然后继续同步。

## 回收站验证（删除/恢复/清空）
1. 在首页打开一条记录详情 -> 删除
2. 到“账户”页 -> 回收站 -> 能看到该记录
3. 点“恢复” -> 记录回到正常列表
4. 回收站点“清空回收站” -> 回收站为空（该操作会在下次同步时上报为 `purge_deleted`）

## 常见问题定位
- 登录后仍提示未登录：
  - 确认后端启动时 `ACCOUNTING_COOKIE_SECURE=0`（本地 http 下 Secure Cookie 不会被浏览器发送）
  - 确认前端通过 dev server 代理访问后端（避免跨域导致 Cookie 不生效）
- 点击“立即同步”报 401：
  - 说明 Cookie 会话未带上或已失效，重新登录再试
- 点击“立即同步”报 403（csrf）：
  - 说明 `csrf_token` Cookie 或 `x-csrf-token` header 缺失/不匹配，重新登录再试
