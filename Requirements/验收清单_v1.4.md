# v1.4 端到端验收清单（多账本体验 + 同步）

目标：验证 v1.4 修复项可复现通过，并确保 v1.3 关键路径不回退。

## 1) 多账本选择不影响全局当前账本（v1.4 新增）

准备：至少存在 2 个账本（账本A、账本B），且两个账本都有记录。

- [ ] 在首页选择“账本A”，确认首页仅展示账本A记录
- [ ] 进入“账户”页，选择账本A+账本B（多选）
- [ ] 返回首页，仍只展示“当前账本（账本A）”的记录（不应变成多账本）
- [ ] 在首页切换到“账本B”，首页数据正确切换（切换功能不失效）
- [ ] 进入“统计/报表”页，数据仍按当前账本展示（不受账户页多选影响）

## 2) 新设备登录后账本自动可见可切换（v1.4 新增）

准备：设备A已登录并存在至少 1 个账本与数据。

- [ ] 设备B使用全新浏览器/无痕模式打开前端，登录同一账号
- [ ] 登录后等待自动同步完成或点击“立即同步”
- [ ] 设备B账本列表可见（无需先新建账本）
- [ ] 设备B可切换账本并正常展示分类/记录

## 3) 账本同步跨端一致性回归（v1.4 重点回归）

准备：设备A、设备B均已登录同一账号。

- [ ] 设备A新建账本C，等待自动同步完成或点击“立即同步”
- [ ] 设备B点击“立即同步”后，可见账本C并可切换
- [ ] 设备A在账本C新增一条记录，设备B同步后可见该记录
- [ ] 设备A切换账本回到账本A，设备B同步后不应把当前账本强行切到其他账本（各端可独立选择当前账本）

## 4) 邀请码管理入口可用（v1.4 回归）

- [ ] 在设置页点击“邀请码管理”，能打开后端管理页（不跳回应用首页）
- [ ] 管理页点击“生成邀请码（7天）”成功，刷新列表可见新增项
- [ ] 未登录直接访问管理页与接口被正确拦截

## 5) v1.3 关键路径不回退（回归）

- [ ] 登录后刷新页面仍保持登录态
- [ ] 写接口缺少 CSRF 返回 403；带 `x-csrf-token` 后成功
- [ ] 设置页同步状态正常展示；点击“立即同步”成功显示 pulled/pushed/version
- [ ] 回收站删除/恢复/清空在多端同步后表现一致
