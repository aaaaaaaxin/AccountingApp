# 家庭记账应用需求文档

## 1. 需求概述
- 一句话描述：开发一个家庭使用的网页版记账应用，支持记录收支、分类管理、账本管理，数据存储在本地。
- 目标用户：家庭用户
- 使用场景：记录家庭日常收支，管理不同类别的花费，支持多账本管理，方便查看收支情况。

## 2. 功能需求
### 2.1 核心功能
- `[ ]` 收支记录：可以添加收入和支出记录，包括金额、日期、备注
- `[ ]` 记录管理：可以编辑和删除已有的收支记录
- `[ ]` 记录列表：可以查看所有历史收支记录（无限滚动）
- `[ ]` 分类管理：对收支进行分类（如餐饮、交通、工资等），预设常用分类
- `[ ]` 账本管理：支持创建和切换多个账本（如生活账本、生意账本等）
- `[ ]` 首页统计：展示本月收入、支出、结余概览
- `[ ]` 搜索功能：支持按备注搜索记录
- `[ ]` 转账记录：支持记录账户间的转账（后续版本v2.0实现）

### 2.2 用户交互
- 用户可以在网页上添加新的收支记录
- 用户可以选择或管理收支分类
- 用户可以编辑或删除已有记录
- 用户可以查看历史记录列表
- 添加记录时可以选择任意日期
- 界面简洁易用，适合家庭用户操作
- 支持标签页切换（支出/收入）
- 支持快捷数字键盘输入金额
- 支持支付方式选择（如微信、支付宝等）
- 支持记录标签（如必要消费、报销等）
- 支持记录模版功能，快速保存和使用常用记录
- 支持"再记"功能，快速连续添加记录
- 支持添加子分类（2级：主分类→子分类）
- 记录列表支持无限滚动加载
- 首页展示本月收支统计概览
- 支持按备注搜索记录

### 2.5 边界规则
- 子分类层级：支持2级（主分类→子分类）
- 金额规则：金额必须>0，支持2位小数，上限999999.99
- 删除分类：提示用户选择（迁移到"其他"或删除记录）
- 删除账本：必须二次确认，确认后删除账本及其所有记录
- 删除单条记录：需要确认
- 账本隔离：各账本数据完全独立
- 模版范围：模版属于当前账本私有
- 权限模型：单用户模式，所有操作权限一致
- 数据容量：单个账本最多10000条记录，最多10个账本，最多50个分类（含子分类）
- 文本长度：备注最多500字符，账本/分类名称最多50字符
- 记录列表：无限滚动加载
- 存储方案：使用IndexedDB进行数据持久化

## 3. 验收标准
- 1. 可以成功添加收入和支出记录，包含金额、日期、备注、分类
- 2. 可以编辑和删除已有的收支记录
- 3. 可以查看所有历史收支记录的列表，支持无限滚动加载
- 4. 可以创建和管理收支分类，预设常用分类
- 5. 可以为任意分类添加子分类（最多2级），可以编辑/删除子分类
- 6. 可以创建新账本、切换当前账本、删除账本（含二次确认）
- 7. 添加记录时可以选择任意日期（年月日），时间默认为当前时间，也可手动调整
- 8. 数据保存在本地，刷新页面后数据不丢失
- 9. 界面简洁清晰，主要功能按钮在首页首屏可见；从打开应用到完成添加一条记录，最多点击3次
- 10. 支持快捷数字键盘输入金额
- 11. 支持选择支付方式
- 12. 支持为记录添加标签
- 13. 支持将当前记录保存为模版，支持从模版快速创建新记录
- 14. 支持"再记"功能，快速连续添加记录
- 15. 删除分类时弹出选项：1) 迁移记录到"其他"分类 2) 同时删除该分类下的所有记录
- 16. 金额输入验证：必须>0，支持2位小数，上限999999.99
- 17. 首页展示本月收入、支出、结余概览
- 18. 支持按备注搜索记录
- 19. 删除单条记录时需要确认
- 20. 数据容量限制：单个账本最多10000条记录，最多10个账本

## 4. 范围边界

- 分类管理功能（含2级子分类）

- 本地数据存储（IndexedDB）

- 标签系统
- 模版功能
- 快速连续记账
- 删除分类/账本/记录的确认机制
- 金额输入验证
- 首页收支统计概览
- 记录搜索功能
- 无限滚动列表
- 数据容量限制

### Out（不包含）
- 云端数据同步
- 账号登录系统
- 复杂的统计报表（后续版本考虑）
- 家庭成员管理
- 数据导入导出（后续版本考虑）

- 多级子分类（超过2级）
- 多用户权限控制
- 批量操作（批量删除/编辑）
- 数据导出功能（v1.1暂不实现）
- 深色/浅色主题切换

## 5. 约束与假设
- 技术栈：前端 React + TypeScript，使用IndexedDB进行数据持久化
- 纯前端应用，不需要后端服务器
- 数据存储在本地浏览器，不需要网络连接
- 目标用户为普通家庭用户，界面需简洁易懂
- MVP版本先实现核心功能，复杂功能后续迭代
- 单用户模式，不区分角色权限
- 各账本数据完全隔离
- 使用IndexedDB进行数据存储

## 6. 风险与回滚
- 风险：本地数据存储可能会因浏览器清除数据而丢失
- 回滚：定期提醒用户备份数据，或提供数据导出功能（后续版本考虑）
- 转账功能暂时不实现，留到v2.0版本
- 数据导出功能暂时不实现，留到后续版本
