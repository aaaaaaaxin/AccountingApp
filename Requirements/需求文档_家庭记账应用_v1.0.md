# 家庭记账应用需求文档

## 1. 需求概述
- 一句话描述：开发一个家庭使用的网页版记账应用，支持记录收支、分类管理，数据存储在本地。
- 目标用户：家庭用户
- 使用场景：记录家庭日常收支，管理不同类别的花费，方便查看收支情况。

## 2. 功能需求
### 2.1 核心功能
- `[ ]` 收支记录：可以添加收入和支出记录，包括金额、日期、备注
- `[ ]` 记录管理：可以编辑和删除已有的收支记录
- `[ ]` 记录列表：可以查看所有历史收支记录
- `[ ]` 分类管理：对收支进行分类（如餐饮、交通、工资等），预设常用分类

### 2.2 用户交互
- 用户可以在网页上添加新的收支记录
- 用户可以选择或管理收支分类
- 用户可以编辑或删除已有记录
- 用户可以查看历史记录列表
- 添加记录时可以选择任意日期
- 界面简洁易用，适合家庭用户操作

## 3. 验收标准
- 1. 可以成功添加收入和支出记录，包含金额、日期、备注、分类
- 2. 可以编辑和删除已有的收支记录
- 3. 可以查看所有历史收支记录的列表
- 4. 可以创建和管理收支分类，预设常用分类
- 5. 添加记录时可以选择任意日期
- 6. 数据保存在本地，刷新页面后数据不丢失
- 7. 界面简洁清晰，主要功能在首页可见，3步内可完成添加记录操作

## 4. 范围边界
### In（包含）
- 收支记录功能
- 分类管理功能
- 本地数据存储
- 网页应用界面

### Out（不包含）
- 云端数据同步
- 账号登录系统
- 复杂的统计报表
- 家庭成员管理
- 数据导入导出

## 5. 约束与假设
- 技术栈：前端 React + TypeScript，使用浏览器本地存储（IndexedDB/LocalStorage）
- 纯前端应用，不需要后端服务器
- 数据存储在本地浏览器，不需要网络连接
- 目标用户为普通家庭用户，界面需简洁易懂

## 6. 风险与回滚
- 风险：本地数据存储可能会因浏览器清除数据而丢失
- 回滚：定期提醒用户备份数据，或提供数据导出功能（后续版本考虑）
